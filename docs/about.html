The modern learning concern

Learning is great, especially when self-interest driven. Yet there is a problem we all face: forgetting what we would like to retain, when memory does not survive the test of time.

Tools have been created to address this. SuperMemo pioneered the field, and many others followed, Anki being one of the most notable. New apps and extensions are regularly being released, which is encouraging. Yet it might seem paradoxical that so many of them, while tackling long-term learning, remain isolated from one another. Most people want to learn, but in different environments, and this fragmentation brings its own set of concerns.

  1. Fragility. Building a learning system inside an app means that system is only as stable as the app itself. If the app is discontinued or no longer maintained, migration becomes painful and years of accumulated learning data may simply be lost.
  2. Duplicated effort. Rather than building on a shared foundation, each tool tends to reimplement what already exists elsewhere. This spreads development effort across isolated solutions that could have focused on what makes each of them unique, and more importantly, on pushing the common ground further rather than rebuilding it from scratch each time, a philosophy that open-source communities have long embraced.
  3. Scattered foundation. There is no stable, common ground for someone who wants to commit to long-term learning. Anki is a great example of what a solid, open and proven tool looks like, yet this raises a natural question: why not just use Anki?
  4. The limits of memorization. Traditional spaced repetition tools excel at retaining atomic knowledge over time. But learning complex topics is fundamentally different. Understanding history, physics, or any field where concepts build on one another requires progressive construction of knowledge. This is a different problem, and one that most traditional tools were not designed to solve. SuperMemo, which pioneered this space, remains largely inaccessible to most learners.

These observations are what motivated the creation of a tool that would address them without adding yet another isolated solution to the mix, but rather one that integrates into what already exists.

One backbone to rule them all

Mycel is an open-source Python backend gathering all of this learning-related functionality and exposing it through a standardized REST API.

It is built around Markdown, storing content as plain text in an SQLite or PostgreSQL database alongside learning data. It can be self-hosted or accessed through a remote instance.

Note that it is in a early develoment stage and that things discussed here may not be yet fully implemented.


Among other things, it handles:

  • Content ingestion - resource gathering, parsing and conversion to Markdown
  • Extraction - user-driven decomposition of learning material into nodes
  • Flashcards - cloze deletion for now, with more types planned
  • Spaced repetition - prioritization logic, FSRS algorithm, and custom scheduling
  • Session logic - review and learning sessions with undo, reschedule, and more
  • Markdown operations - formatting, outlining, splitting
  • Configuration - dynamic config across implementations
  • Import/export - JSON to and from SQLite or PostgreSQL
  • Extensibility - custom algorithms, resource types, templates, and more
  • Built to last - long-term versioning and backward compatibility across implementations
  • REST API - agnostic to any client

The goal is to make it as simple as possible for implementations to interact with, while preserving flexibility for those who need it. Markdown is the current content format, but it is not deeply rooted into the architecture, leaving room for other markup languages if needed.
